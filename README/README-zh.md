# WebDown 使用说明

把网页真正保存下来，随时离线查看。

## 它能做什么

WebDown 是一个 Chrome 浏览器扩展，帮你把网页和它依赖的资源（图片、样式、字体、脚本等）打包成 ZIP 文件，下载到本地后可以直接离线浏览。

支持两种模式：
- **保存当前页** — 把你正在看的页面完整保存下来
- **整站抓取** — 从一个起始页面出发，沿着链接自动抓取多个页面和资源

## 安装

1. 在 [Releases](https://github.com/webdown-a/webdown/releases) 页面下载最新的 `webdown-extension-vX.X.X.zip`
2. 解压到一个固定目录（不要删除）
3. 打开 Chrome，地址栏输入 `chrome://extensions`
4. 右上角开启「开发者模式」
5. 点击「加载已解压的扩展程序」，选择刚才解压的文件夹
6. 工具栏会出现 WebDown 图标，安装完成

## 使用方法

### 保存当前页

1. 打开你想保存的网页
2. 点击工具栏的 WebDown 图标
3. 确认模式为「保存当前页」
4. 点击「开始」
5. 完成后 ZIP 文件自动下载，解压即可离线浏览

### 整站抓取

1. 打开目标站点的任意页面
2. 点击 WebDown 图标，切换到「整站抓取」模式
3. 设置起始 URL（默认为当前页面地址）
4. 根据需要调整以下选项：
   - **抓取深度** — 从起始页向外跟踪几层链接（0 = 只抓起始页）
   - **页面范围** — 限制抓取范围
     - 仅当前域名：只抓同一域名下的页面
     - 同主站/子域名：包含子域名（如 blog.example.com）
     - 任意域名：跟到哪抓到哪（可配合白名单/黑名单使用）
   - **指定可抓取域名** — 任意域名模式下，限定只抓这些域名
   - **排除域名** — 跳过不想抓的域名（优先级最高）
5. 点击「开始」

### 资源控制

- **允许下载外部资源** — 是否下载其他域名的图片、样式等
- **下载第三方脚本** — 是否包含外部 JS 文件
- **下载音视频资源** — 是否包含视频和音频文件

### 动态页面

如果目标页面是通过 JavaScript 动态渲染的（如 React、Vue 构建的页面）：
- 勾选「等待渲染后抓取」
- 适当增加「渲染等待」时间（毫秒）

### 性能调节

- **串行模式** — 一次只下载一个文件，更稳定但较慢
- **并发下载数** — 同时下载的文件数，越大越快，但对目标服务器压力越大

### 任务控制

抓取过程中你可以随时：
- **暂停** — 停止抓取，已下载的内容会立即打包为 ZIP 下载
- **取消** — 停止抓取，已下载的内容会立即打包为 ZIP 下载

### 分批下载

当资源量很大时，WebDown 会自动分批打包：
- 每批达到一定数量或大小后，自动生成一个 ZIP 并下载（如 `site-part1.zip`、`site-part2.zip`）
- 已下载的批次不会丢失，即使后续出现问题
- 资源量较小时，仍然生成一个 ZIP 文件，和以前一样

### 下载结果

- ZIP 文件包含完整的网页和资源，解压后双击 `index.html` 即可浏览
- 如果需要在浏览器中打开多个 HTML 文件，可以用本地服务器：
  ```
  python3 -m http.server 8080
  ```
  然后访问 `http://localhost:8080`

## 适合什么场景

- 收藏的网页怕链接失效，提前保存一份
- 出差、坐飞机前把资料离线保存
- 做调研时把参考页面按项目归档
- 把教程、文档保存成本地知识库
- 给团队分享页面内容，不怕原文被修改或删除

## 使用建议

| 场景 | 建议 |
|------|------|
| 静态站点 | 提高并发数，速度更快 |
| 动态站点（SPA） | 开启渲染等待，适当增加等待时间 |
| 大型站点 | 先用小深度测试，确认没问题再加大 |
| 网络不稳定 | 先暂停，等网络恢复后已下载内容不会丢失 |
